# 移动端优化 PRD

> 审计设备: iPhone 13 Mini (375x812 viewport)
> 审计日期: 2026-04-01
> 审计站点: https://chunzhao.vercel.app

## 一、用户旅程分析

### 典型路径：小红书 → 校招助手

```
小红书笔记 → 点击链接 → 首页落地
  → 浏览首页价值主张 (5s 判断是否有价值)
  → 点击「开始找工作」进入职位库
  → 筛选/搜索职位 → 点击「导入申请池」
  → 弹出登录提示 → 跳转登录页
  → 注册/登录 → 返回职位库
  → 导入职位 → 进入看板管理
  → (可选) 查看笔试真题 / 升级付费
```

### 当前旅程断点分析

| 断点位置 | 问题 | 影响程度 |
|---------|------|---------|
| 首页 → 职位库 | CTA 按钮有效，路径通畅 | 低 |
| 职位库 → 导入 | 需登录但提示不够明显，用户可能困惑 | 中 |
| 登录页 | 表单定位过低（top: 419px），首屏大量空白浪费 | **高** |
| 看板/进度页 → 未登录 | 整页空白仅一个「去登录」按钮，无引导价值 | **高** |
| 笔试真题 → 外链 | 链接跳转到牛客网，用户流失不可控 | 中 |

## 二、页面逐页分析

### 1. 首页 (/)

**实测数据:**
- 导航栏高度: 57px
- 汉堡菜单按钮: 36x36px (低于 44x44 最小触控标准)
- H1 字号: 30px, 行高: 37.5px
- 正文字号: 14px
- 页面内边距: 16px 左右
- CTA「开始找工作」: 142x44px (合格)
- CTA「查看我的看板」: 133x45px (合格)

**当前问题:**

1. **汉堡菜单触控区域不足**: 36x36px，低于 Apple HIG 推荐的 44x44px 最低标准。用户在拥挤的地铁等场景下难以准确点击。
2. **首屏信息层级缺失**: 首屏只有一个 tagline + 两个按钮。缺少社会证明（用户数、学校 logo 等），对从小红书跳转过来的陌生用户说服力不足。
3. **功能介绍卡片需滚动才能看到**: 三张功能卡（一键导入、看板管理、关键提醒）全部在 fold 以下，首次访客无法快速理解产品能力。
4. **底部 CTA 区域视觉权重不足**: 深色背景区域的「查看升级计划」按钮引导用户付费，但从小红书来的新用户尚未注册，此处不应是付费引导，而应是注册引导。
5. **没有底部导航栏**: 移动端完全依赖顶部汉堡菜单进行导航，增加了操作成本。常用功能（职位库、看板、真题）应在底部 tab 直达。

**目标效果:**
- 首屏即传达核心价值 + 社会证明
- 触控目标全部 >= 44x44px
- 添加移动端底部导航栏

**具体改动方案:**

```
A. 汉堡菜单按钮
   当前: w-9 h-9 (36x36px)
   目标: min-w-[44px] min-h-[44px], 可保留图标尺寸但扩大点击热区 (padding)

B. 首屏结构重组
   当前顺序: badge → h1 → description → 2 CTAs
   目标顺序: badge → h1 (缩至 26px) → 简短一句话 → 主 CTA → 社会证明条
   社会证明条: "已有 3,000+ 同学在使用" + 3-4 个学校 logo 缩略图

C. 底部导航栏 (新增)
   结构: 4 tab — 首页 | 职位库 | 看板 | 我的
   样式: fixed bottom-0, h-[56px], bg-white/90 backdrop-blur, border-t
   所有页面的 main content 需要增加 pb-[72px] 避免被遮挡

D. 底部深色区 CTA 调整
   未登录: "免费注册，开始管理" → /login
   已登录: "查看升级计划" → /pricing (保持现有)
```

### 2. 职位库 (/jobs)

**实测数据:**
- 搜索框: 328x37px, 字号 14px
- 筛选下拉: 328x36px (偏矮，触控边缘)
- 筛选标签「全部」: 48x29px (严重不足 44px 标准)
- 筛选标签「26届热门春招」: 117x29px (高度不足)
- 「导入申请池」按钮: 110x32px (高度不足)
- 「查看原JD」链接: 68x16px (严重不足，极难点击)
- 职位标题字号: 16px, 行高 24px
- 「加载更多」按钮: 133x41px (接近合格)

**当前问题:**

1. **筛选标签高度严重不足**: 全部 tag 按钮高度仅 29px，远低于 44px 标准。密集排列时误触率极高。
2. **「查看原JD」链接几乎无法点击**: 68x16px 的链接在移动端是灾难性的触控目标。
3. **「导入申请池」按钮偏小**: 110x32px，高度仅 32px。作为核心转化按钮，应该更突出。
4. **职位标题过长被截断**: 很多职位标题包含十几个岗位名称，在移动端被 text-overflow: ellipsis 截断，用户无法获取关键信息。
5. **搜索框和筛选器占据过多首屏空间**: 搜索 + 城市下拉 + 企业下拉 + 标签行 = 约 200px，在 812px 屏幕上占了 25%，首屏仅能看到 1-2 张卡片。
6. **标签溢出行为**: 4 个筛选 tag 在 375px 宽度下需要横向滚动，但没有滚动提示（如渐变遮罩或箭头），用户不知道可以滑动。
7. **无粘性筛选栏**: 用户滚动查看职位后，想修改筛选条件需要滚回顶部。

**目标效果:**
- 所有可交互元素 >= 44px 触控高度
- 筛选栏吸顶，职位列表可无限滚动
- 职位卡片信息密度优化

**具体改动方案:**

```
A. 筛选标签尺寸
   当前: h-[29px] px-3 text-sm
   目标: min-h-[36px] px-4 text-sm, 外层容器 min-h-[44px] (含 padding)
   标签行改为 overflow-x-auto + snap-x，右侧加 gradient mask 提示可滑动

B. 「查看原JD」链接
   当前: 纯文字链接 68x16px
   目标: 改为 ghost button 样式, min-h-[36px] px-3, 带 border
   或者将其与「导入申请池」并排为等高按钮

C. 「导入申请池」按钮
   当前: h-[32px] px-3
   目标: h-[40px] px-4, font-medium

D. 筛选栏吸顶
   搜索框 + 筛选下拉 + 标签 → 包裹在 sticky top-[57px] 容器中
   加 bg-white shadow-sm 分隔

E. 职位卡片优化
   标题: 限制 max 2 行 (line-clamp-2)，超出显示「等 N 个岗位」
   公司名称: 加大字号至 15px，加粗
   标签: 保持当前尺寸但确保间距 >= 8px
   底部操作栏: 两个按钮等宽排列，高度 40px

F. 筛选下拉高度
   当前: h-[36px]
   目标: h-[44px]
```

### 3. 看板页 (/board) — 未登录态

**当前问题:**

1. **整页空白浪费**: 仅显示一个图标 + 一行文字 + 一个按钮，位于页面正中。从小红书跳转到此页的用户会立即跳出。
2. **没有产品预览**: 用户不知道登录后能看到什么。无动力注册。
3. **「去登录」按钮过小且不醒目**: 在大片空白中，一个小按钮缺乏吸引力。

**目标效果:**
- 未登录用户也能感受到看板的价值
- 提供清晰的注册动力

**具体改动方案:**

```
A. 增加看板预览截图
   在登录提示上方，展示一张看板界面的 mock 截图（或半透明带模糊效果）
   截图展示: 待评估(3) → 笔试(2) → 面试(1) → Offer(1) 的列布局

B. 增加价值说明
   当前: "请先登录以使用此功能"
   目标: "看板帮你追踪每个岗位的投递进度\n从网申到 Offer，一目了然"

C. CTA 按钮升级
   当前: 小尺寸「去登录」
   目标: 全宽按钮 "免费注册，开始管理" + 下方小字 "已有账号？直接登录"
   按钮: w-full h-[48px] bg-blue-600 text-white rounded-xl
```

### 5. 定价页 (/pricing)

**实测数据:**
- 页面标题: "选择你的计划", 居中
- 两张卡片: 免费版 + 完整版(¥9.9)
- 免费版 CTA「免费开始」: 带边框的 ghost button
- 完整版 CTA「立即解锁」: 蓝色实心 button

**当前问题:**

1. **首屏被免费版卡片占满**: 完整版卡片（核心付费转化）需要滚动才能看到。用户可能看到免费版就点击离开了。
2. **免费版卡片信息太少**: 只有 3 条特性，但占据了大量垂直空间。
3. **完整版价格锚定不明显**: ¥9.9 永久 是极具吸引力的价格，但没有划线原价对比、没有「限时」紧迫感。
4. **无社会证明**: 没有显示有多少人已经付费升级。
5. **「推荐」badge 位于两卡之间的分割线上**: 在移动端视觉上容易被忽略。

**目标效果:**
- 完整版卡片优先展示或至少与免费版同屏可见
- 价格锚定更清晰

**具体改动方案:**

```
A. 卡片顺序调整（移动端）
   方案1: 完整版在上，免费版在下（推荐）
   方案2: 保持顺序但压缩免费版卡片高度

B. 免费版卡片压缩
   当前: 特性列表 + 大按钮 + 底部说明
   目标: 水平紧凑布局，特性用逗号分隔一行展示
   或将免费版做成折叠的 accordion，默认收起

C. 价格锚定优化
   添加划线价: "原价 ¥29.9" → "¥9.9 / 永久"
   添加限时标签: "春招特惠" 或 "限时价格"

D. 社会证明
   卡片下方: "已有 1,200+ 同学升级完整版"
```

### 6. 笔试真题页 (/exam)

**实测数据:**
- 页面标题: "笔试真题库", 带图标, 橙色主题
- 搜索框 + 公司 tag 筛选 + 真题卡片列表
- 每张卡片: 公司图标 + 标题 + 标签 + 「前往」外链

**当前问题:**

1. **卡片操作区偏小**: 「前往」链接是整张卡片唯一的交互点，但视觉面积小。
2. **外链跳转无预期管理**: 点击「前往」直接跳转到牛客网，用户离开校招助手。没有提示用户即将离开站点。
3. **公司筛选标签横向溢出**: 8 个公司 tag（腾讯、字节跳动、华为等）在 375px 宽度下需要横向滚动，无滚动提示。
4. **标签样式与职位库不一致**: 这里用橙色系，职位库用蓝色系，缺乏统一感。
5. **真题内容太少**: 仅 10 条结果且全是技术方向，非技术岗用户（财务、HR、市场）会感到无用。

**目标效果:**
- 标签滚动可发现
- 外链跳转有提示
- 整卡可点击而非仅「前往」

**具体改动方案:**

```
A. 卡片改为整卡可点击
   当前: 卡片内嵌「前往」link
   目标: 整张卡片 cursor-pointer, onClick 打开链接
   「前往」改为右侧箭头图标提示

B. 外链跳转提示
   首次点击外链时显示 toast: "即将跳转到牛客网"
   或在卡片底部小字标注: "资源来自牛客网"

C. 标签行滚动提示
   overflow-x-auto + 右侧渐变遮罩 (linear-gradient to right, transparent → white)

D. 增加非技术方向真题
   行测/申论/案例分析/行业知识 等分类
```

### 7. 登录页 (/login)

**实测数据:**
- 表单卡片 top: 419px (在 812px 屏幕上偏下，首屏 52% 是空白)
- 输入框: 279x41px, 字号 14px
- 登录/注册 tab: 137x36px
- 提交按钮「登录」: 279x40px
- H1 "登录校招助手": 20px

**当前问题:**

1. **首屏 50% 是空白**: 表单卡片位于 Y=419px，用户打开页面看到大片空白 + 一个小标题，必须滚动才能看到输入框。这是移动端登录页的严重问题。
2. **没有第三方登录**: 仅邮箱+密码登录。对移动端用户而言，微信登录几乎是必备的（目标用户是大学生，微信渗透率 99%）。
3. **没有忘记密码入口**: 表单中无 "忘记密码?" 链接。
4. **tab 切换（登录/注册）触控高度**: 36px，低于 44px 标准。
5. **提交按钮宽度与输入框一致但偏矮**: 40px 高度，在长输入框对比下显得不够突出。
6. **无返回导航**: 用户从职位库被跳转到登录页后，没有明显的返回路径（只有顶部 logo 可以回首页）。

**目标效果:**
- 表单首屏完整可见
- 支持微信快捷登录
- 操作路径清晰

**具体改动方案:**

```
A. 表单位置上移
   当前: 表单卡片有大量上方空白
   目标: 卡片 mt-[80px] 或更少，确保输入框 + 按钮完整在首屏内
   可在表单上方加产品 logo + 一句 slogan

B. 登录/注册 tab 尺寸
   当前: h-[36px]
   目标: h-[44px]

C. 提交按钮强化
   当前: h-[40px]
   目标: h-[48px], rounded-xl, font-semibold text-base

D. 增加微信登录
   在邮箱表单下方增加分割线 "—— 或 ——"
   微信登录按钮: w-full h-[48px] bg-[#07C160] text-white
   （需后端对接微信 OAuth）

E. 增加忘记密码链接
   在密码输入框右下方: "忘记密码?" text-sm text-blue-600

F. 增加返回按钮
   在页面标题左侧增加 ← 返回箭头, 点击 history.back()
```

## 三、全局移动端问题

### 3.1 缺少底部导航栏 (Bottom Tab Bar)

**现状**: 所有页面导航依赖顶部汉堡菜单，这是移动端最大的 UX 缺陷。

**方案**: 添加固定底部导航栏

```
结构:
  首页(/) | 职位库(/jobs) | 看板(/board) | 我的

样式:
  position: fixed
  bottom: 0
  height: 56px + safe-area-inset-bottom
  background: rgba(255,255,255,0.9)
  backdrop-filter: blur(12px)
  border-top: 1px solid #f1f5f9

图标: lucide-react
  首页: Home
  职位库: Search
  看板: LayoutGrid / Kanban
  我的: User (包含登录/个人中心)
```

### 3.2 汉堡菜单按钮触控不足

**所有页面通用问题**: 36x36px 汉堡按钮。

**修复**: 视觉尺寸可保持，但添加 `p-1` padding 使触控热区达到 44x44px。

### 3.3 无 Safe Area 适配

**现状**: 页面底部没有针对 iPhone 刘海屏 / home indicator 的 safe area 适配。

**修复**: 底部导航和底部按钮添加 `pb-[env(safe-area-inset-bottom)]`。

### 3.4 页面间缺少过渡

**现状**: 页面切换是硬切，没有过渡动画。

**建议**: 至少为核心路径（首页→职位库→登录）添加简单的 fade 或 slide 过渡。

### 3.5 无下拉刷新

**现状**: 职位库页面没有 pull-to-refresh 功能，移动端用户的本能操作无法使用。

## 五、关键指标

### 核心转化指标

| 指标 | 当前基线(估) | 目标 | 衡量方式 |
|------|-------------|------|---------|
| 首页 → 职位库 点击率 | - | >= 40% | 埋点: CTA click / page view |
| 职位库 → 注册 转化率 | - | >= 15% | 埋点: 「导入申请池」触发登录 / 职位库 UV |
| 登录页注册完成率 | - | >= 60% | 注册成功 / 登录页 UV |
| 注册 → 首次导入 | - | >= 50% | 首次 import_job / 注册用户数 |
| 付费转化率 | - | >= 5% | 付费 / 注册 7 天内 |

### 体验指标

| 指标 | 当前基线(估) | 目标 | 衡量方式 |
|------|-------------|------|---------|
| 看板/进度空白页跳出率 | >= 80% | <= 40% | 页面停留 < 3s 的占比 |
| 登录页放弃率 | >= 50% | <= 25% | 打开登录页但未提交的占比 |
| 移动端首页停留时间 | < 10s | >= 20s | GA / 自有埋点 |
| 职位库筛选使用率 | - | >= 30% | 使用筛选 / 职位库 PV |

### 监控方式

1. **Vercel Analytics**: 页面级 PV、UV、跳出率
2. **自有埋点** (推荐 PostHog 或 Umami): 按钮点击、转化漏斗、用户路径
3. **A/B 测试**: 登录页改版前后对比、定价页卡片顺序对比
